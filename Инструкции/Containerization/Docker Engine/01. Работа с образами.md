### 01. Работа с образами (Images)
```bash
#скачать образ из реестра (Docker Hub)
docker pull <имя_образа> 

#посмотреть список скачанных образов
docker images || docker image ls 

#собрать образ
docker build -t <имя:тег> -f path/Dockerfile
-t #присваивает образу имя и тег.
-f #указывает путь к Dockerfile
--no-cache #заставляя выполнять все инструкции заново
--add-host "имя_хоста:IP-адрес" #добавляет запись в /etc/hosts контейнера

#удалить образ
docker rmi <имя_образа> 

#присвоить образу новый тег (обычно нужно перед загрузкой в свой реестр).
docker tag <старое_имя> <новое_имя>
```
### 02. Жизненный цикл контейнеров
```bash
#создать и запустить новый контейнер
docker run <образ>
-d #запустить в фоне (detach).
-p 8080:80 #пробросить порт (локальный порт:порт в контейнере).
--name my_nginx #задать имя контейнеру. 
--restart=unless-stopped #режим перезапуска, всегда, кроме случаев, когда администратор явно выполнил stop
-w /workspace #делает /workspace рабочей директорией
-v /home/data:/app/data #пробросить папку (том).
-e MY_VAR=value #задать переменную окружения.
--entrypoint tail <образ> -f /dev/null #замена точки входа
```
```bash
#управление состоянием
docker ps #показать запущенные контейнеры.
docker ps -a #показать все контейнеры (включая остановленные).
docker stop <имя_или_id>  #остановить контейнер (мягко).
docker kill <имя_или_id>  #остановить принудительно (немедленно).
docker start <имя_или_id>  #запустить ранее созданный и остановленный контейнер.
docker restart <имя_или_id>  #перезагрузить контейнер.
```
```bash
#очистка
docker rm <имя_или_id> #удалить остановленный контейнер.
docker rm -f <имя_или_id> #удалить запущенный контейнер (с принудительной остановкой).
```
### 03. Работа внутри запущенного контейнера
```bash
#выполнить команду внутри работающего контейнера
docker exec -it <имя> <команда>
-i #интерактивный режим (оставить STDIN открытым).
-t #выделить псевдо-терминал (TTY).

#посмотреть логи
docker logs <имя> 
-f #следить за логами в реальном времени (как tail -f).

#скопировать файл из контейнера на хост или наоборот.
docker cp <имя>:<путь_внутри> <путь_снаружи> 
```
### 04. Системные команды и очистка места
```bash
#показать, сколько места занимает Docker
docker system df

#удалить все неиспользуемые данные (остановленные контейнеры, сети, висячие образы)
docker system prune
-f #пропускает запрос подтверждения перед удалением
-a #еще и все образы, которые не привязаны ни к одному контейнеру
```
