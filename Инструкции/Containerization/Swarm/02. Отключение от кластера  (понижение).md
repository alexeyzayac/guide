### 01. Отключение воркера.
Необходимо на ноде воркера выполнить :
```bash
docker swarm leave
docker swarm leave --force #принудительный выход
```
После того как узел покинул кластер, он  отображаться в списке узлов со статусом `Down` на мэнеджере. Чтобы полностью удалить эту запись из конфигурации swarm, используйте команду:
```bash
docker node rm <ИМЯ_УЗЛА>
```
---
### 02. Отключение менеджера (понижение).
Для начала необходимо понизить менеджера до воркера, выполнив эту команду на любом другом работающем менеджере кластера:
```bash
docker node demote <ИМЯ_УЗЛА>
```
Затем необходимо переключитесь на терминал самого удаляемого узла и выполнить:
```bash
docker swarm leave
docker swarm leave --force
```
После того как узел покинул кластер, он  отображаться в списке узлов со статусом `Down` на мэнеджере. Чтобы полностью удалить эту запись из конфигурации swarm, используйте команду:
```bash
docker node rm <ИМЯ_УЗЛА>
```
---
### 03. Для полного расформирования кластера
Если вы хотите **полностью остановить и удалить весь swarm-кластер**, вы можете использовать флаг `--force`. Это проигнорирует предупреждения и заставит узел покинуть swarm, даже если он является менеджером (на воркерах при этом контейнеры не остановятся).
```bash
docker swarm leave --force
```
* **Когда это уместно:** Этот способ подходит только для ситуаций, когда вы точно знаете, что swarm больше не будет использоваться, например, в тестовом или одноузловом кластере
