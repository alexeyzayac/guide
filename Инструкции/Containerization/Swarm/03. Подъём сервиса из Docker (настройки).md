###  01. Разворот сервиса командой
```bash
docker service create \
  --name nginx \
  --publish published=80,target=80 \
  --constraint "node.role==worker" \
  --replicas 3 \
  --limit-cpu 0.5 \
  --limit-memory 256M \
  --reserve-cpu 0.25 \
  --reserve-memory 128M \
  alexeyzayac/custom-nginx:1.0.0
```
---
###  02. Работа с сервисом
#### Посмотрите текущие сервисы
```bash
docker service ls
```
#### Управление выводом задач сервиса
```bash
#распределение реплик по узлам
docker service ps nginx

#показывать только работающие задачи
docker service ps nginx \
  --filter "desired-state=running" 

#показать только остановленные задачи
docker service ps nginx \
  --filter "desired-state=shutdown"

#показать историю конкретной реплики
docker service ps nginx \
  --filter "name=nginx.1" 

#Альтернативы для «чистоты», чтобы вывести только нужные поля
docker service ps nginx \
  --filter "desired-state=running" \
  --format "table {{.Name}}\t{{.Node}}\t{{.CurrentState}}"
```
#### Детальная информация о сервисе
```bash
docker service inspect --pretty nginx
```
#### Логи сервиса
```bash
docker service logs nginx
```

###  03. Распределение задач по узлам
Чтобы «заставить» Swarm перебалансировать реплики между обоими рабочими узлами, нужно либо вызвать пересоздание задач, либо явно указать политику размещения. Самый безопасный и быстрый способ — **принудительно обновить сервис**:
```bash
docker service update --force nginx
```
#### Альтернативные методы
**Ручное масштабирование:**
```bash
docker service scale nginx=0
docker service scale nginx=4
```
**Использование placement preferences:**
```bash
docker service update \
  --placement-pref-add spread=node.labels.node \
  nginx
```

### 04. Остановка сервиса
Если нужно полностью удалить сервис:
```bash
docker service rm nginx
```
Все контейнеры, связанные с сервисом, будут остановлены и удалены со всех нод.

#### Остановка реплик только на определенной ноде
##### Изменение доступности ноды:
```
#перевести ноду в режим Drain (задачи будут перемещены)
docker node update --availability drain <имя воркера>

#вернуть обратно в Active
docker node update --availability active <имя воркера>
```
##### Использование ограничения при обновлении сервиса:
```bash
#добавление ограничения
docker service update --constraint-add 'node.hostname!=<имя воркера>' nginx

#удаление ограничения
docker service update --constraint-rm 'node.hostname!=<имя воркера>' nginx
```
##### Использование placement preferences (распределение с учетом меток)
```bash
# Добавляем метку на ноду
docker node update --label-add region=west vm-node-worker-1
docker node update --label-add region=east vm-node-worker-2

# Обновляем сервис, чтобы он работал только на нодах с меткой region=west
docker service update --constraint-add 'node.labels.region==west' nginx
```
##### Временное отключение ноды от кластера (пауза)
Если вы хотите, чтобы нода не принимала новые задачи, но текущие остались работать, используйте:
```bash
docker node update --availability pause <имя воркера>
	```
