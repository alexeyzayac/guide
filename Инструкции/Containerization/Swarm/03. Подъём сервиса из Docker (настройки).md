###  01. Разворот сервиса командой
```bash
docker service create \                # создать сервис в swarm-кластере
  --name nginx \                       # имя сервиса
  --publish published=80,target=80 \   # проброс порта 80(хост) на 80(контейнер)
  --constraint "node.role==worker" \   #запускать реплики только на worker-узлах
  --replicas 3 \                       # количество реплик (контейнеров)
  --limit-cpu 0.5 \                    # максимум 0.5 ядра на контейнер
  --limit-memory 256M \                # лимит оперативной памяти
  --reserve-cpu 0.25 \                 # гарантированно 0.25 ядра
  --reserve-memory 128M                # гарантированно 128 МБ
  alexeyzayac/custom-nginx:1.0.0
```

###  02. Работа с сервисом
#### Посмотрите текущие сервисы
```bash
docker service ls
```
#### Управление выводом задач сервиса
```bash
#распределение реплик по узлам
docker service ps nginx

#показывать только работающие задачи
docker service ps nginx \
  --filter "desired-state=running" 

#показать только остановленные задачи
docker service ps nginx \
  --filter "desired-state=shutdown"

#показать историю конкретной реплики
docker service ps nginx \
  --filter "name=nginx.1" 

#Альтернативы для «чистоты», чтобы вывести только нужные поля
docker service ps nginx \
  --filter "desired-state=running" \
  --format "table {{.Name}}\t{{.Node}}\t{{.CurrentState}}"
```
#### Детальная информация о сервисе
```bash
docker service inspect --pretty nginx
```
#### Логи сервиса
```bash
docker service logs nginx
```

###  03. Распределение задач по узлам
Чтобы «заставить» Swarm перебалансировать реплики между обоими рабочими узлами, нужно либо вызвать пересоздание задач, либо явно указать политику размещения. Самый безопасный и быстрый способ — **принудительно обновить сервис**:
```bash
docker service update --force nginx
```
#### Альтернативные методы
**Ручное масштабирование:**
```bash
docker service scale nginx=0
docker service scale nginx=4
```
**Использование placement preferences:**
```bash
docker service update \
  --placement-pref-add spread=node.labels.node \
  nginx
```
