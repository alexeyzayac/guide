# Команда `fsck` – проверка целостности файловой системы

**`fsck`** (от *File System Check*) – стандартная утилита Unix/Linux для проверки и **исправления** повреждённых файловых систем. Работает на низком уровне: анализирует суперблок, inode, битовые карты, каталоги и журналы. Это инструмент «последней надежды» при серьёзных сбоях.

> ⚠️ **Важно:** `fsck` **всегда** должна запускаться на **размонтированной** файловой системе. Исключение – корневой раздел в режиме восстановления (single user mode) или при монтировании `ro`. Использование на смонтированной ФС **гарантированно повредит данные**.

## ⚙️ Быстрый старт
```bash
# Принудительная проверка /dev/sdb1 (демонтируем!)
sudo umount /dev/sdb1
sudo fsck -y /dev/sdb1

# Только просмотр ошибок (без исправлений) для корневого раздела
sudo fsck -n /dev/sda2

# Проверить все ФС из fstab (кроме корня), показать прогресс
sudo fsck -C -R
```
## 📖 Полезные опции
|Опция|Длинная форма|Действие|
|---|---|---|
|`-y`|–|На все вопросы отвечать «да» (автоисправление)|
|`-n`|–|Отвечать «нет» (только просмотр, ничего не менять)|
|`-f`|`--force`|Принудительная проверка, даже если ФС «чистая»|
|`-c`|–|Поиск плохих блоков (для ext2/ext3, не для современных дисков)|
|`-C`|`--progress`|Отображать прогресс-бар (для ext4, xfs)|
|`-t`|`--type=тип`|Явно указать тип ФС (ext4, vfat, xfs, btrfs)|
|`-M`|–|Не проверять, если устройство не смонтировано (для скриптов)|
|`-R`|–|Пропустить корневую ФС (при проверке всех из fstab)|
|`-V`|`--verbose`|Подробный вывод (показывает, какая конкретная утилита вызывается)|
|`-A`|–|Проверить все ФС из `/etc/fstab` (обычно используется в скриптах загрузки)|
|`-l`|–|Заблокировать устройство (исключить параллельные проверки)|
|`-r`|–|Статистика по восстановленным inode (устарело)|
> 💡 Для разных ФС `fsck` вызывает специализированные утилиты: `fsck.ext4`, `fsck.xfs`, `fsck.vfat`, `fsck.btrfs` и т.д. Обёртка `fsck` сама определяет тип по суперблоку или `/etc/fstab`.
## 🧪 Примеры команд с пояснениями
```bash
# 1. Принудительная проверка /dev/sdb1 (сначала размонтировать)
sudo umount /dev/sdb1
sudo fsck -y /dev/sdb1

# 2. Только просмотр ошибок (без исправлений) для раздела Windows (vfat)
sudo fsck -n -t vfat /dev/sdc1

# 3. Проверить все ФС, кроме корневой, с прогресс-баром
sudo fsck -C -R -A

# 4. Принудительная проверка ext4 в режиме только чтения (без изменений)
sudo fsck -n -f /dev/mapper/ubuntu--vg-root

# 5. Поиск плохих блоков на старом диске (ext3) – ОЧЕНЬ долго
sudo fsck -c /dev/hda1
```
