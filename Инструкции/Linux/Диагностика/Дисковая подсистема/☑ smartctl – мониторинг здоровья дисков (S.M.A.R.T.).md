# `smartctl` – мониторинг здоровья дисков (S.M.A.R.T.)

**`smartctl`** (из пакета `smartmontools`) – утилита для работы с системой самодиагностики **S.M.A.R.T.** (Self‑Monitoring, Analysis and Reporting Technology). Позволяет оценить «здоровье» HDD, SSD, NVMe, предсказать отказ и вовремя заменить накопитель, предотвратив потерю данных.

> ⚠️ **Требует прав root** (или соответствующих `sudo`). Многие команды могут выполняться без остановки работы диска, но длительные тесты (`-t long`) создают нагрузку.

## ⚙️ Быстрый старт
```bash
# 1. Найти все диски с поддержкой SMART
sudo smartctl --scan

# 2. Получить общую информацию о диске (модель, серийный номер, версия прошивки)
sudo smartctl -i /dev/sda

# 3. Быстрая проверка здоровья (PASSED / FAILED)
sudo smartctl -H /dev/sda

# 4. Просмотреть все атрибуты SMART (таблица сырых значений)
sudo smartctl -A /dev/sda

# 5. Короткий самотест (обычно 1-2 минуты)
sudo smartctl -t short /dev/sda

# 6. Длительный самотест (часы, полное сканирование поверхности)
sudo smartctl -t long /dev/sda

# 7. Посмотреть результаты самотестов
sudo smartctl -l selftest /dev/sda
```
## 📖 Полезные опции (таблица)
|Опция|Длинная форма|Назначение|
|---|---|---|
|**Информация**|||
|`-i`|`--info`|Общие сведения о диске (модель, прошивка, ёмкость, поддержка SMART)|
|`-H`|`--health`|Состояние здоровья (PASSED/FAILED) на основе атрибутов|
|`-A`|`--attributes`|Таблица всех атрибутов SMART (сырые, нормализованные, пороговые значения)|
|`-c`|`--capabilities`|Возможности диска (поддерживаемые тесты, функции)|
|**Тестирование**|||
|`-t short`|`--test=short`|Запустить короткий самотест (обычно 1-2 минуты)|
|`-t long`|`--test=long`|Длительный самотест (полное сканирование поверхности)|
|`-t conveyance`|`--test=conveyance`|Тест транспортировки (проверяет повреждения при перевозке)|
|`-t select,N-M`|–|Тест выбранного диапазона LBA-секторов (экономит время)|
|`-X`|`--abort`|Прервать выполняющийся самотест|
|`-l selftest`|`--log=selftest`|Показать журнал результатов самотестов|
|`-l error`|`--log=error`|Показать журнал ошибок (CRC, чтения, записи)|
|`-l directory`|–|Список доступных журналов SMART|
|**Управление**|||
|`-s on` / `-s off`|`--smart=on` / `off`|Включить/выключить сбор SMART (обычно и так включён)|
|`-o on` / `-o off`|`--offlineauto=on` / `off`|Автоматический сбор офлайн-данных|
|`-S on` / `-S off`|`--saveauto=on` / `off`|Автосохранение атрибутов|
|**Формат вывода**|||
|`-j`|`--json`|Вывод в JSON (удобно для скриптов и систем мониторинга)|
|`-H -j`|–|Только состояние здоровья в JSON|
|`-A -j`|–|Атрибуты в JSON|
|`--scan -j`|–|Список дисков в JSON|

> 💡 Многие опции можно комбинировать, например `sudo smartctl -H -A /dev/sda`.

## 🧪 Расширенные примеры команд
```bash
# 1. Просмотр ошибок диска (включая историю)
sudo smartctl -l error /dev/sda

# 2. Включить SMART, если он отключен (редко, но бывает)
sudo smartctl -s on /dev/sda

# 3. Показать атрибуты только с ненулевыми raw-значениями (фильтр через grep)
sudo smartctl -A /dev/sda | grep -v " 0$" | grep -E "^ [0-9]|^  5|^197|^198|^  9|^194"

# 4. Короткий тест и ожидание его завершения
sudo smartctl -t short /dev/sda && sleep 120 && sudo smartctl -l selftest /dev/sda

# 5. JSON для мониторинга (например, в Prometheus)
sudo smartctl -H -j /dev/sda | jq '.smart_status.passed'

# 6. Тест выбранного диапазона (первые 10 GB)
sudo smartctl -t select,0-10000000 /dev/sda
```
## 📊 Ключевые атрибуты SMART и их расшифровка
| Атрибут (ID) | Название в smartctl      | Что означает                                                                              | На что обратить внимание                                                                                                                            |
| ------------ | ------------------------ | ----------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| **5**        | `Reallocated_Sector_Ct`  | Количество секторов, которые диск нашел поврежденными и переназначил из резервной области | Любое ненулевое значение — плохой признак. Оно показывает, что диск деградирует.                                                                    |
| **197**      | `Current_Pending_Sector` | Количество секторов, которые вызывают подозрения, но еще не были переназначены            | Ненулевое значение — повод для беспокойства. Если значение станет больше нуля и не изменится после проверки `badblocks`, диск может выйти из строя. |
| **198**      | `Offline_Uncorrectable`  | Количество секторов, которые не удалось прочитать или восстановить.                       | Ненулевое значение — критично. Означает реальную потерю данных.                                                                                     |
| **9**        | `Power_On_Hours`         | Общее количество часов работы диска с момента его выпуска                                 | Позволяет оценить физический износ. Для обычных HDD ресурс может составлять 50 000+ часов, но это сильно зависит от модели и условий.               |
| **194**      | `Temperature_Celsius`    | Текущая температура диска в градусах Цельсия                                              | В идеале должна быть ниже 50°C. Перегрев значительно сокращает срок службы.
