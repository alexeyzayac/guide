`lsof` (от англ. **L**ist **O**pen **F**iles) — мощная утилита в Unix/Linux, которая показывает все открытые файлы в системе. В мире Linux «файл» — это не только обычные файлы, но и каталоги, сокеты (сетевые соединения), каналы (pipes), устройства (например, `/dev/sda`) и даже динамические библиотеки.
```bash
lsof [опции] [имя_файла]
```
#### Зачем нужна
- Узнать, какой процесс использует определённый файл или каталог (чтобы понять, почему не удаётся размонтировать диск).
- Посмотреть, какие сетевые порты заняты и каким процессом (альтернатива `netstat -tulpn`).
- Найти процессы, которые держат файл удалённым (`rm` не срабатывает).
- Отследить утечки файловых дескрипторов.

## Пример вывода
- **COMMAND** – имя процесса
- **PID** – идентификатор
- **USER** – владелец
- **FD** – файловый дескриптор (cwd – текущий каталог, mem – отображённый файл, 1u – дескриптор 1 для чтения/записи)
- **TYPE** – тип (REG – обычный файл, DIR – каталог, IPv4/IPv6 – сокет)
- **NAME** – имя файла или адрес:порт
## Часто используемые опции
|Опция|Назначение|
|---|---|
|`-u пользователь`|только файлы, открытые указанным пользователем|
|`-p PID`|только процессы с заданным PID|
|`-i`|сетевые файлы (IPv4/IPv6)|
|`-i :порт`|что слушает/использует порт|
|`-i @адрес`|соединения с указанным IP|
|`+D /путь`|рекурсивно найти файлы, открытые внутри `/путь`|
|`-t`|выдать только PID (удобно для скриптов)|
|`-c команда`|процессы, начинающиеся с имени команды|
|`-d fd`|показать файлы с конкретным дескриптором (например, `1` — stdout)|
## Практические примеры
```bash
#Какой процесс использует файл /var/log/syslog
lsof /var/log/syslog

#кто слушает порт 80
lsof -i :80

#все сетевые соединения текущего пользователя
lsof -i -u $(whoami)

#процессы, открывшие файлы в каталоге /mnt/usb
lsof +D /mnt/usb

#показать только PID процессов, которые используют файл
lsof -t /path/to/locked_file

#что открыл процесс с PID 5678
lsof -t /path/to/locked_file

#все TCP-сокеты
lsof -i tcp
```
