# `lsof` – список всех открытых файлов в системе

**`lsof`** (от *List Open Files*) – мощная утилита Unix/Linux, показывающая все открытые файлы. В Linux «файл» – это не только обычные файлы, но и каталоги, сокеты (сетевые соединения), каналы (pipes), устройства (`/dev/sda`) и даже динамические библиотеки. Незаменима для диагностики «занятых» файлов, поиска процессов, слушающих порты, и обнаружения утечек дескрипторов.

> ⚠️ Часть информации требует прав root (например, чужие процессы). Запускайте с `sudo` для полной картины.

## ⚙️ Быстрый старт
```bash
# 1. Какой процесс использует файл (например, не даёт размонтировать диск)
lsof /var/log/syslog

# 2. Кто слушает порт 80 (альтернатива netstat -tulpn | grep 80)
sudo lsof -i :80

# 3. Все сетевые соединения текущего пользователя
lsof -i -u $(whoami)

# 4. Процессы, открывшие файлы в каталоге /mnt/usb (рекурсивно)
lsof +D /mnt/usb

# 5. Только PID процессов, использующих файл (удобно для kill)
lsof -t /path/to/locked_file

# 6. Что открыл процесс с PID 5678
lsof -p 5678
```
## 📖 Полезные опции
|Опция|Длинная форма|Назначение|
|---|---|---|
|**Фильтрация по объектам**|||
|`-u пользователь`|`--user`|Только файлы, открытые указанным пользователем|
|`-p PID`|–|Только процессы с заданным PID (можно несколько: `-p 1234,5678`)|
|`-c команда`|–|Процессы, чьё имя начинается с `команда` (например `-c ssh`)|
|`+D /путь`|–|Рекурсивно найти файлы, открытые внутри `/путь` (осторожно, медленно для больших деревьев)|
|`-d fd`|–|Показать файлы с конкретным дескриптором (например `-d 1` – stdout, `-d 2` – stderr)|
|**Сетевые фильтры**|||
|`-i`|–|Показать только сетевые файлы (IPv4/IPv6)|
|`-i :порт`|–|Что слушает или использует порт (например `-i :443`)|
|`-i @адрес`|–|Соединения с указанным IP (например `-i @192.168.1.1`)|
|`-i tcp` / `-i udp`|–|Только TCP или только UDP|
|`-i tcp:80`|–|Конкретный протокол и порт|
|`-i 4` / `-i 6`|–|Только IPv4 или IPv6|
|**Управление выводом**|||
|`-t`|–|Выдать только PID (без заголовков, удобно для скриптов)|
|`-F`|–|Вывод в формате, пригодном для разбора (например `-F p` для PID)|
|`-w`|–|Подавить предупреждения (например, о недоступных точках монтирования)|
|`-n`|–|Не преобразовывать IP-адреса в имена (ускоряет вывод)|
|`-P`|–|Не преобразовывать номера портов в имена сервисов (ускоряет)|
|**Разное**|||
|`-l`|–|Показывать UID вместо имени пользователя (полезно, если имя не резолвится)|
|`+L`|–|Показать файлы с указанным количеством ссылок (например `+L1` – файлы с нулевыми ссылками, удалённые, но открытые)|
|`-a`|–|Объединяет условия по **И** (AND), по умолчанию – ИЛИ.|

> 💡 Комбинация `-n -P` ускоряет `lsof` на больших системах, отключая DNS-резолвинг и преобразование портов.

## 🧪 Расширенные примеры команд
```bash
# 1. Найти все открытые файлы в /home (осторожно, может быть много)
sudo lsof +D /home | less

# 2. Процессы, которые держат удалённые (deleted) файлы (занимают место)
sudo lsof +L1 | grep deleted

# 3. Все соединения с конкретного IP
sudo lsof -i @10.0.0.5

# 4. Только слушающие сокеты (LISTEN)
sudo lsof -i -s TCP:LISTEN

# 5. Получить PID процесса, использующего порт 3306 (для остановки)
sudo lsof -t -i :3306

# 6. Комбинация условий: процесс с именем nginx И сетевые файлы
sudo lsof -c nginx -i

# 7. Вывести PID и имя процесса для всех TCP-соединений (скриптовый режим)
sudo lsof -F p c -i tcp
```
