`du` (disk usage) — это стандартная утилита в Linux и Unix-системах, предназначенная для оценки и отображения дискового пространства, занимаемого файлами и каталогами. В отличие от `df`, которая показывает свободное место на всём разделе, `du` позволяет узнать, сколько места занимает конкретная папка или файл, что делает её идеальным инструментом для поиска «пожирателей» дискового пространства.
```bash
du [ОПЦИИ] [ФАЙЛ/КАТАЛОГ]
```
#### Полезные опции:
**`-h`** - Выводит размеры в читаемом формате
**`-s`** - Выводит **только общий итог** для каждого аргумента, не показывая размеры подкаталогов
**`-c`** - Добавляет в конце общую сумму по всем переданным аргументам.
**`-S`** - Для каталогов не включает в их размер размеры подкаталогов.
**`-a`** - Показывает размеры **всех файлов**, а не только каталогов.
**`--exclude=""`**  - исключить файлы или каталоги
**`-x, --one-file-system`** - Не переходить на другие файловые системы (например, смонтированные удалённые диски).
**`-X, --exclude-from=FILE`** - Исключает файлы, соответствующие шаблонам, перечисленным в файле `FILE`
**`-d, --max-depth=N`** - Ограничивает глубину рекурсивного обхода до `N` уровней. Например, `--max-depth=1` покажет размеры только каталогов первого уровня, что очень удобно для быстрой оценки.
**`-l`** - Учитывает размер жёстких ссылок несколько раз. Без этой опции `du` посчитает размер файла только один раз.
#### Примеры команд:
```bash
#показывает общий размер каталога `/home/user`
du -sh /home/user

#поиск самых больших подкаталогов в корне
du -h --max-depth=1 / 2>/dev/null | sort -hr | head -n 5

#анализ текущего каталога с сортировкой
du -ah --max-depth=1 | sort -h

#показывает размер каждой из указанных папок и общую сумму
du -shc /home /var
```
