`fsck` (от англ. **F**ile **S**ystem **C**heck) — стандартная утилита Unix/Linux для проверки целостности файловой системы и исправления найденных ошибок. Она работает на низком уровне с суперблоком, inode, битовыми картами и каталогами.
```bash
fsck [опции] [устройство]
```
Если не указывать устройство, `fsck` проверит все файловые системы из `/etc/fstab` (кроме тех, где `pass` = 0).
#### Когда нужен fsck?
- Система не загружается или монтирует разделы только в режиме «только чтение».
- При работе появились ошибки ввода-вывода, файлы стали отображаться как «????????».
- После сбоя питания или нештатного выключения (ext4 может восстановить журнал автоматически, но иногда требуется ручной fsck).
- Периодическая профилактика (например, каждые 30 монтирований или определённое время — настройка tune2fs).

## Популярные опции
| Опция    | Действие                                                          |
| -------- | ----------------------------------------------------------------- |
| `-y`     | на все вопросы отвечать «да» (автоисправление)                    |
| `-n`     | отвечать «нет» — только просмотр ошибок, ничего не менять         |
| `-f`     | принудительная проверка, даже если файловая система «чистая»      |
| `-c`     | проверить плохие блоки (для старых дисков, ext2/3)                |
| `-C`     | показывать прогресс-бар (для некоторых ФС, например ext4)         |
| `-t тип` | задать тип файловой системы (ext4, vfat, xfs и т.п.)              |
| `-M`     | не проверять, если устройство не смонтировано                     |
| `-R`     | пропустить корневую файловую систему (при проверке всех из fstab) |
## Примеры использования
```bash
#принудительно проверить /dev/sdb1 (размонтировав предварительно)
sudo umount /dev/sdb1
sudo fsck -y /dev/sdb1

#только посмотреть, что не так (без исправлений)
sudo fsck -n /dev/sda2
```
