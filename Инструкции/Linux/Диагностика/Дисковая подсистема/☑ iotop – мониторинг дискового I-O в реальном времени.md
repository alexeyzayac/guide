# `iotop` – мониторинг дискового I/O в реальном времени

**`iotop`** – интерактивная утилита для Linux, показывающая **какие процессы** и с какой интенсивностью читают или пишут данные на диск. Работает по принципу `top`, но отслеживает не CPU/память, а дисковую активность. Незаменима для поиска «виновнkика» высокой нагрузки на диск.

> ⚠️ **Требует прав root** (или `CAP_SYS_ADMIN`), так как читает данные из `/proc/<pid>/io` и работает с ядерными счётчиками.

## ⚙️ Популярные команды
```bash
sudo iotop -o                     # только процессы, реально выполняющие I/O
sudo iotop -d 2                   # интервал обновления 2 секунды
sudo iotop -u mysql               # активность только пользователя mysql
sudo iotop -p 1234                # отслеживать конкретный PID
sudo iotop -botq -d 5 >> iotop.log # пакетный режим с временными метками
```
## 📖 Полезные опции
|Опция|Длинная форма|Назначение|
|---|---|---|
|`-o`|`--only`|Показывать только процессы, выполняющие I/O (полезно для сужения списка)|
|`-d`|`--delay`|Интервал обновления в секундах (дробные, например `0.5`)|
|`-b`|`--batch`|**Пакетный режим** – неинтерактивный вывод (для скриптов, записи в файл)|
|`-a`|`--accumulated`|Показывать **накопленные** объёмы I/O (с момента запуска), а не текущую скорость|
|`-q`|`--quiet`|Уменьшить количество выводимой информации (может использоваться с `-b`)|
|`-t`|`--timestamp`|Добавить временную метку к каждой строке (полезно в пакетном режиме)|
|`-u`|`--user`|Показать активность только для указанного пользователя|
|`-P`|`--processes`|Показывать только **процессы**, а не все потоки (по умолчанию показываются потоки)|
|`-p`|`--pid`|Отслеживать только указанный PID (можно несколько: `-p 1234 -p 5678`)|
|`-k`|`--kilobytes`|Использовать килобайты вместо человеко-читаемых единиц|
|`-h`|`--help`|Справка|

> 💡 `-q` можно использовать несколько раз (`-qq`), чтобы убрать заголовки и итоговые строки.
## 🧪 Примеры команд с пояснениями
```bash
# 1. Интерактивный режим, только активные процессы, обновление каждые 2 с
sudo iotop -o -d 2

# 2. Пакетный режим с временными метками и минимальным выводом (для лога)
sudo iotop -botq -d 5 >> /var/log/disk_activity.log

# 3. Удалить из лога строки с нулевой активностью
cat disk_activity.log | grep -v "0.00 B/s"

# 4. Мониторинг конкретного процесса и его дочерних потоков
sudo iotop -p 1234 -a

# 5. Сравнение накопленного I/O для нескольких пользователей
sudo iotop -a -u www-data -u mysql -u nobody
```
## 🖥️ Как читать вывод `iotop`

### Верхняя часть (общая статистика системы)

- **Total DISK READ** / **WRITE**: Суммарная скорость чтения/записи **между процессами и подсистемой ядра** (все процессы в сумме).
- **Current DISK READ** / **WRITE**: Фактическая скорость на уровне **физического устройства** (может отличаться из-за кэширования, сливания запросов).
### Основная таблица (столбцы)

| Столбец        | Значение                                                                                      |
| -------------- | --------------------------------------------------------------------------------------------- |
| **TID**        | Идентификатор потока (если включён режим потоков) или PID процесса (опция `-P`)               |
| **PRIO**       | Приоритет I/O (например `be/4` – best-effort, класс 4; `rt/0` – realtime)                     |
| **USER**       | Владелец процесса                                                                             |
| **DISK READ**  | Скорость чтения за интервал                                                                   |
| **DISK WRITE** | Скорость записи за интервал                                                                   |
| **SWAPIN**     | Процент времени, потраченный на подкачку из swap                                              |
| **IO**         | Процент времени ожидания операций ввода-вывода (высокое значение – процесс блокирован диском) |
| **COMMAND**    | Имя процесса/команда                                                                          |

> `IO` – самый важный столбец: если он близок к 100%, процесс почти всё время ждёт диск.

## 🎮 Интерактивные клавиши (в режиме `iotop`)

Во время работы `iotop` можно динамически менять отображение:

|Клавиша|Действие|
|---|---|
|**← / →**|Изменить столбец для сортировки|
|**r**|Обратный порядок сортировки|
|**o**|Переключить режим `--only` (показывать только активные процессы)|
|**p**|Переключить режим **процессы / потоки** (аналог `-P`)|
|**a**|Переключить **текущая скорость / накопленный объём** (аналог `-a`)|
|**q**|Выход|
|**h**|Показать помощь|

> В пакетном режиме (`-b`) интерактивные клавиши недоступны.
