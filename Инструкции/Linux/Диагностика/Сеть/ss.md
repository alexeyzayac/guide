 это современная, высокопроизводительная команда для детального анализа сетевых соединений. Она пришла на смену устаревшей `netstat` и стала стандартным инструментом для системных администраторов и инженеров по сетевым технологиям
```bash
ss [опции] [фильтр]
```
#### Фильтрация по протоколу
- `ss -t` — отобразить только **TCP** сокеты
- `ss -u` — отобразить только **UDP** сокеты
- `ss -x` — отобразить **UNIX domain** сокеты
- `ss -4` или `ss -6` — отобразить сокеты только для **IPv4** или **IPv6**
#### Управление выводом
- `ss -l` - показать **только слушающие (LISTEN)** сокеты (порты, ожидающие входящих подключений)
- `ss -a` -  показать **все сокеты**: как слушающие, так и установленные соединения
- `ss -n` - **не преобразовывать** номера портов в имена сервисов (показывать `:22` вместо `:ssh`). **Эта опция критически важна для ускорения работы команды
- `ss -r` - **преобразовывать** IP-адреса в доменные имена через DNS
- `ss -p` - показать **идентификатор и имя процесса (PID)**, который использует данный сокет. **Требует прав суперпользователя (root) для полной информации**
- `ss -s` - вывести **сводную статистику** по всем протоколам (количество сокетов в различных состояниях)
- `ss -e` - показать **расширенную информацию** о сокете 
- `ss -i` - показать **внутреннюю информацию TCP** (размер окна, RTT, состояние таймеров и другие параметры)
### Примеры использования
```bash
#увидеть все сетевые подключения (TCP, UDP, RAW, UNIX)
ss -tunap

#посмотреть, какие порты слушает ваш сервер
ss -tln

#получить краткую сводку по сокетам
ss -s
```
