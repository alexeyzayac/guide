Чтобы кластер работал корректно через публичные IP, на всех ВМ должны быть открыты следующие порты (в облачном firewall / security group):
- **2377/tcp** – для управления Swarm (только между менеджером и воркерами)
- **7946/tcp и 7946/udp** – для связи между узлами (gossip)
- **4789/udp** – для overlay-сетей (VXLAN)
### Создание Swarm кластера (1 мастер, 2 ноды)
#### 1. На машине менеджера (`пример:` IP: 46.21.245.210)
Инициализируем Swarm и указываем advertise-адрес (публичный IP менеджера, чтобы воркеры могли подключиться):
```bash
docker swarm init --advertise-addr 46.21.245.210
```
После успешной инициализации вы увидите команду для добавления рабочих нод.
#### 2. На машинах воркерах 1 и 2 выполните команду из пункта выше
Это подключит ноду к машине менеджеру (пример ниже):
```bash
docker swarm join --token SWMTKN-1-... 46.21.245.210:2377
```
#### 3. Проверка списка нод:
На **машине менеджера** выполните:
```bash
docker node ls
```
Обратите внимание:
- Менеджер имеет статус `Leader` (если он единственный менеджер).
- Воркеры не имеют `MANAGER STATUS`.
- Все ноды должны быть `Ready` и `Active`.
